## Tech Stack — Notable Changes

### Cilium / Gateway API
- Cilium 1.15+ ships native Gateway API support as stable (not experimental).
  `GatewayClass` → `Gateway` → `HTTPRoute` works without feature flags.
  Hubble UI available as a separate Helm chart (`cilium/hubble`).
- Kubernetes Gateway API v1.1 (May 2024) promoted `HTTPRoute` and `GRPCRoute`
  to GA. `TLSRoute` remains experimental. Watch for `BackendLBPolicy` GA —
  it is the official path for weighted routing and health checks.
- **Action:** confirm Cilium version pinned in k3s lab matches Linode LKE
  availability when we provision the hosted cluster.

### Chainguard
- Chainguard Python images now include Python 3.12 and 3.13 variants.
  The `cgr.dev/chainguard/python:latest` tag tracks the most recent stable.
  For reproducible Enterprise builds, pin to a digest not a tag.
- Chainguard introduced `wolfi-base` as a public image for teams that need
  a shell for debugging — still minimal, but not as stripped as the default
  python image. Useful for Enterprise troubleshooting without compromising
  the production image.

### freenginx
- freenginx continues active development under Maxim Dounin. No formal
  foundation or LTS release announced yet as of this brief.
  Monitor [freenginx.org](https://freenginx.org) for governance announcements.
- For Kubernetes: moot — we use Cilium Gateway API. For Proxmox lab reverse
  proxy: freenginx 1.27.x is a drop-in nginx replacement.

### Rocky Linux / Debian
- Rocky Linux 9.3 is current stable. RESF continues active development.
  No governance concerns observed.
- Debian 12 (Bookworm) is current stable. Debian 13 (Trixie) in testing.

## Market Signals — Mock OIDC / Identity Testing Space

### Competitive landscape (as of this brief)
- **navikt/mock-oauth2-server** — Most starred OSS mock OIDC (3k+ stars).
  Still no hosted offering. Recent issues requesting Entra ID claim shapes
  and OBO flow — both unimplemented. This is our strongest differentiator gap.
- **axa-group/oauth2-mock-server** — npm-only, incomplete PKCE, no hosted.
  Limited Python ecosystem presence. Low activity in recent months.
- **WireMock Cloud** — HTTP mock, not OIDC-aware. No JWT signing. Not a
  direct competitor but sometimes cited in the same breath.
- No new entrant has launched a hosted mock OIDC SaaS in the $29–49/month
  range. The gap remains open.

### Community signals
- Stack Overflow: recurring questions about "mock Entra ID tokens in CI"
  with no clean answer. The accepted answers reference Keycloak (heavy)
  or navikt (no hosted). This is the exact pain point mock-idp solves.
- GitHub Discussions on azure-sdk repos: teams asking for lightweight
  Entra ID token generation for integration tests. No official Microsoft
  recommendation beyond "use a real tenant."
- Reddit r/devops: thread about CI pipelines depending on external IdPs —
  common complaint, no good solution named.

### Standards activity
- RFC 9449 (DPoP — Demonstrating Proof of Possession) gaining traction in
  financial services. Not a near-term roadmap item but worth tracking.
- FAPI 2.0 (Financial-grade API Security Profile) references are appearing
  in enterprise RFPs. Relevant if we pursue financial services Enterprise
  customers.

## Open Source Governance Watch

- **F5 / nginx:** No reversal of the governance decisions that caused
  Dounin's fork. freenginx continues as the community alternative.
- **Canonical / Ubuntu:** Snap continues as default for some packages.
  No major new controversies this week. Governance concern remains at
  steady-state — not escalating, not resolving.
- **Oracle:** No new open source hostile actions this week. Standing
  policy: Oracle stays out of our stack.
- **HashiCorp / Vault:** BSL license change (2023) is settled. OpenTofu
  is the community fork of Terraform. For Vault specifically: BSL applies
  to Vault 1.14+. Vault 1.13 and below remain MPL-2.0. OpenBao is the
  community fork of Vault — worth monitoring as the "follow the developer"
  choice if HashiCorp further restricts Vault. Not urgent for our use case
  (Vault sidecar integration is an Enterprise feature, not yet built).
